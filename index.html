How it works

Single Custom Hook

All API calls and state management live inside one custom hook — useWeather. Components are purely presentational: they receive data as props and only handle rendering.

This separation keeps every component lean and the logic testable in isolation.

// hooks/useWeather.js const useWeather = () => { const [weather, setWeather] = useState(null); const [forecast, setForecast] = useState([]); const fetchCity = async (city) => { // /weather + /forecast }; return { weather, forecast, fetchCity }; };

Gradient Mapping

OpenWeatherMap returns condition codes like 800 (clear sky) or 2xx (thunderstorm). These codes are mapped to a color palette inside helpers.js to produce the background gradient.

The 5-day forecast is aggregated from 3-hour interval data into daily highs and lows, also inside helpers.

// helpers.js export const getGradient = (code) => { if (code === 800) return CLEAR_COLORS; if (code >= 200 && code < 300) return STORM_COLORS; // clouds, rain, snow… };

// get started

Run it locally

Requires Node.js v18+ and the Expo Go app on your Android or iOS device. Add your OpenWeatherMap API key to a .env file and you're good to go.

$ git clone https://github.com/jshobhit13/weather-prophesy.git
$ cd weather-prophesy
$ npm install
# Add EXPO_PUBLIC_WEATHER_API_KEY to .env
$ npx expo start --clear
✓ Metro bundler running · scan QR with Expo Go
